# 银行信贷决策支持报告

## 概述

为银行普惠部/信贷审批部生成信贷决策支持报告。**核心使命：回答"该不该放贷"而非"值不值得买"。**

### 背景与场景

银行普惠部面临的核心困境：
- 支行推荐的企业常获政府重点扶持（奖补、专精特新），但财务指标不佳（营收利润双降、负债上升）
- 审批部门坚持传统风控逻辑——所有财务指标必须正向增长，否则拒绝
- 业务部门反复口头协调风控，缺乏客观数据支撑
- 行内未建立模型体系，仍停留在专家经验规则层面
- 缺乏足够样本数据（存量客户仅几千家），尤其对细分科技类小企业
- 对企业所处行业赛道、未来经营情况缺乏科学预判

### 报告定位

| 维度 | 本报告（信贷决策支持） | 传统投资分析报告 |
|------|------|------|
| **读者** | 信贷审批官/普惠部客户经理 | 投资者/分析师 |
| **核心问题** | 该不该放贷？ | 值不值得买？ |
| **核心指标** | 信用评分/还款能力/现金跑道 | ROE/PB/DCF估值 |
| **时间取向** | 未来12个月预测为主 | 历史财务分析为主 |
| **财务逻辑** | 生存优先（现金跑道>利润率） | 盈利优先 |
| **数据权重** | 财务40%+行业20%+政策15%+技术15%+团队10% | 财务数据占70%+ |
| **输出物** | 信用评分+贷款建议+预警指标 | 投资评级+目标价 |

### 触发关键词

银行信贷审批、贷款决策、普惠金融、科技金融风控、信贷决策支持、企业还款能力评估、银行授信报告、"这家企业能不能贷"、"普惠部审批"。

## 核心创新框架

### 🔴 框架一：反周期信贷逻辑（理论内核）

传统审批逻辑要求所有财务指标正向增长，这把科技型中小企业系统性地排除在外。反周期信贷逻辑按企业类型差异化设标：

| 企业类型 | 核心判断指标 | 财务容忍度 | 典型行业 | 判定关键词 |
|------|------|:--:|------|------|
| **研发投入型** | IP增量+补贴连续性+融资能力 | 高（允许营收/利润双降） | 生物医药、半导体、新材料 | 专利增长/研发占比>15%/融资多轮 |
| **订单驱动型** | 中标金额+预收款+产能匹配 | 中（营收季节性波动正常） | 建筑施工、IT服务、设备制造 | 中标活跃/预收款占比高/产能利用率 |
| **流量平台型** | 用户/交易量增速+融资能力 | 高（允许长期亏损） | SaaS、电商、平台经济 | 用户增速>50%/交易量/GMV增长 |
| **传统制造型** | 财务指标+产能利用率+现金流 | 低（传统风控逻辑适用） | 传统制造业、批发零售 | 营收/利润/现金流均为正/固定资产占比高 |

**判定流程（🔴 强制——报告第一章即需输出）：**

```
企业数据 → 招聘岗位类型×中标项目类型×IP聚类 → 识别实际赛道 → 匹配企业类型 → 选择适用的信贷逻辑
```

**为什么这很重要：** 银行审批部门"一刀切"拒绝科技型企业的根因，是用传统制造型企业的风控标准去衡量研发投入型企业。本框架将不同类型企业的适用标准前置化，业务部门不再需要"反复口头协调"——报告本身就是客观依据。

### 🔴 框架二：企业分类自动判定规则

通过以下数据自动判定企业类型：

| 判定指标 | 研发投入型 | 订单驱动型 | 流量平台型 | 传统制造型 |
|------|:--:|:--:|:--:|:--:|
| 研发人员占比（从招聘岗位推断） | >30% | <10% | 15-30% | <5% |
| 知识产权年增量 | >5件/年 | 1-5件 | 不定 | 0-1件 |
| 政府补贴特征 | 科技/工信部门 | 无特定 | 不定 | 无/少量 |
| 中标项目类型 | 技术服务 | 工程/设备 | 平台/软件 | 制造/贸易 |
| 毛利率（如有） | >50% | 15-30% | >50% | 10-25% |

### 🔴 框架三：结构化信用评分卡（0-100分）

| 评分模块 | 权重 | 核心指标 | 数据来源 |
|------|:--:|------|------|
| **财务健康** | 30% | 营收趋势/毛利率/现金跑道(月)/负债结构 | Doris tb_annual_report_2 + 上市公司数据 |
| **行业前景** | 20% | 行业增速/政策导向强度/生命周期阶段 | Web搜索 |
| **政府背书** | 15% | 补贴连续性(年)/补贴层级/专精特新/高新/纳税信用 | Doris补贴+标签+税务 |
| **技术创新** | 15% | 专利数/有效发明专利/软著/研发活跃度 | STI + Doris IP指数 |
| **管理团队** | 10% | 创始人履历/团队稳定性/融资能力/跨实体关联 | Web搜索+融资记录 |
| **市场验证** | 10% | 中标金额/客户增长/融资轮次/估值趋势 | 中标+融资事件 |
| **合计** | **100%** | — | — |

**评级→信贷建议映射（🔴 报告核心输出之一）：**

| 评分区间 | 评级 | 信贷建议 | 额度建议 | 期限建议 |
|:--:|:--:|------|------|------|
| ≥80 | A | **建议批准**——多维数据交叉验证支持增长预期，即使部分财务指标为负 | 标准额度 | 1-3年 |
| 65-79 | B | **有条件批准**——需补充担保/降低额度/缩短期限/增加贷后频次 | 标准额度×0.7 | ≤1年 |
| 50-64 | C | **进一步尽调**——建议补充实地调研/上下游访谈/行业专家咨询 | 待尽调后定 | — |
| 35-49 | D | **审慎**——需强担保强抵押+显著提高利率+严格贷后 | 标准额度×0.3 | ≤6个月 |
| <35 | E | **不建议**——多维数据均不支撑 | — | — |

### 🔴 框架四：前瞻性营收预测模型（三大交叉验证）

通过三条独立数据线索，构建"企业未来12个月经营预测三角"：

| 预测线索 | 数据来源 | 方法论 | 置信度 |
|------|------|------|:--:|
| **中标→营收推演** | `ads_bid_win_candidate_out` | 中标金额 × 行业平均执行周期 → 未来12月营收预测区间 | 中-高 |
| **招聘→产能推演** | `mcp_data_query_search_recruitment` | 招聘岗位×数量×薪资→业务方向→未来6-12月产能变化→营收上限 | 中 |
| **融资→现金跑道推演** | `tb_company_financing` + tb_annual_report_2 | 最新融资额 ÷ 月均烧钱率 → 现金跑道(月) → 是否需要续贷 | 高 |

**三线交叉验证规则：**
- 两条及以上线索指向同一方向 → 高置信度预测
- 仅一条线索有数据 → 中等置信度，标注不确定性
- 三条线索方向矛盾 → 低置信度，建议进一步尽调

**三情景营收预测（报告标配）：**

| 情景 | 假设 | 12个月营收预测 | 概率 |
|------|------|------|:--:|
| 乐观 | 中标全部执行+新产品如期上市+融资到账 | 上行区间 | 20% |
| 基准 | 70%中标执行+正常业务增速+现有现金支撑 | 基准区间 | 50% |
| 悲观 | 30%中标执行+业务停滞+无新融资 | 下行区间 | 30% |

### 🔴 框架五：政策信号量化评分

将政府认可转化为可量化的信用增强因子：

| 维度 | 评分标准 | 最高分 |
|------|------|:--:|
| **补贴连续性** | 连续3年+ = 3分 / 2年 = 2分 / 1年 = 1分 / 无 = 0分 | 3 |
| **补贴层级** | 国家级 = 5分 / 省级 = 3分 / 市级 = 2分 / 区级 = 1分 | 5 |
| **专精特新** | 国家级 = 5分 / 省级 = 3分 / 市级 = 2分 / 创新型中小企业 = 1分 | 5 |
| **高新技术企业** | 国家级高新 = 3分 / 科技型中小企业 = 1分 | 3 |
| **纳税信用** | A级 = 2分 / B级 = 1分 / 其他 = 0分 | 2 |
| **政府项目中标** | 有政府项目中标 = 2分 / 无 = 0分 | 2 |
| **合计** | — | **20分** |

**政策信号信用增强规则：** 政策评分≥12分 → 财务指标准入标准可下浮10-15%（如营收增速要求从+5%降至-5%）；政策评分≥16分 → 可下浮15-20%。

## 工作流

### Phase 0：企业类型预判（🔴 每次必执行）

在数据采集前，先通过企业名称和简要背景判断预期属于哪种类型，以调整后续数据采集策略：

- **研发投入型：** 重点采集IP、补贴、融资数据
- **订单驱动型：** 重点采集中标、预收款、产能数据
- **流量平台型：** 重点采集融资、用户增速、市场数据
- **传统制造型：** 重点采集财务、抵押物、现金流数据

## 🔴 报告质量铁律

### 格式严禁

| 禁止项 | 示例 | 原因 |
|------|------|------|
| 字母编号章节 | `## A.1` | 破坏统一结构 |
| 版本标记 | `V1`、`v1.0` | 报告是独立文档 |
| 迭代前缀 | `【新增】`、`★新增` | 暴露后加痕迹 |
| 装饰分隔符 | `═══`、`███` | 非专业格式 |
| 日期重复 | 两处以上报告日期 | 残留痕迹 |

### 内容必须

| 要求 | 说明 |
|------|------|
| 🔴 第一章即输出信贷决策结论+企业类型判定 | 不是"投资结论"，是"建议批准/有条件批准/进一步尽调/审慎/不建议" |
| 🔴 信用评分卡为第二章必含 | 0-100分+各模块得分+评级→信贷建议映射 |
| 🔴 前瞻性营收预测为第25章必含 | 三情景12个月预测+三线交叉验证置信度 |
| 🔴 政策信号量化评分+信用增强规则 | 第28章必含 |
| 🔴 贷后风险预警指标体系 | 第38章必含7项预警指标+阈值 |
| 🔴 企业类型判定+信贷逻辑选择 | 第10章必含，第39章深度论证 |
| 每表必配"分析"段落（100-300字） | 展开推理链，不只是一句话 |
| 每5-8表插入"关键发现"callout | 报告≥15处 |
| 多源数据交叉验证推演 | 中标×招聘×财务×补贴多线交叉 |
| 财务数据标注来源 | Doris tb_annual_report_2标注"企业自行申报税务数据，未经审计" |

## 注意事项

### 与 enterprise-deep-dive-report 的关系

| 维度 | enterprise-deep-dive-report | 本技能 |
|------|------|------|
| **服务对象** | 投资者 | 银行信贷审批 |
| **数据采集** | 32类MCP+≥5次Web+≥8张Doris | 复用+7项信贷专项+≥7次Web+≥10张Doris |
| **报告结构** | 7篇52章 | 7篇43章+增强16章+附录3章=62章 |
| **核心框架** | PB-ROE/DCF估值 | 信用评分卡+反周期信贷逻辑+前瞻预测 |
| **可以共享数据采集** | ✅ MCP/Doris/Web搜索的数据可复用 | ✅ |

### 小微企业数据缺失处理

当Doris无财务数据时（营收<1亿/员工<100人/成立<2年）：
- 信用评分卡中财务健康权重从30%降至20%，行业前景从20%升至25%，政府背书从15%升至20%
- 前瞻性预测更依赖中标和招聘数据（而非财务趋势外推）
- 报告行数目标自动降至≥800行

### 🔴 章节编号铁律

统一数字编号1-62，禁止A-L字母索引。目录与正文章节名完全一致。

### 🔴 表格格式铁律

- 数据对比型表格内置"解读"列
- 汇总型表格必须有"合计"行
- 表格必须用markdown分隔线 `|---|---|---|`（否则PDF不渲染）

### PDF生成陷阱

- `generate_pdf` content参数有~30KB限制，大报告用terminal Python heredoc
- `markdown.markdown()` 必须显式传 `extensions=['tables', 'fenced_code']`
- 表格边框：`border-spacing:0` + 每格独立 `border-right/border-bottom` + 外层`border:3px solid`
- 禁止封面页、页眉页脚、红框突出等视觉升级

### 用户偏好

用户徐生沟通极简。报告完成后直接发送MEDIA文件+简短汇总。批量执行模式下列出优化项后直接全部执行，不逐一确认。

### 🔴 主智能体定稿铁律

`delegate_task` 仅用于Phase 1数据采集和Phase 1B Web研究搜索。Phase 2报告定稿+Phase 3 PDF生成始终由主智能体完成。子智能体写报告会丢失深度（实测958行 vs 原版2053行）。
