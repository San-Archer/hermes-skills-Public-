# 反模式：禁止让 delegate_task 子智能体写最终报告

## 实测数据

| 企业 | 子智能体"终极版" | 主智能体累积版 | 丢失深度 |
|------|:--:|:--:|:--:|
| 凌度智能 | 958行 | 2,053行 | -53% |
| 联信高新 | 731行 | 1,882行 | -61% |
| 莱迪生物 | 672行 | 1,563行 | -57% |

**用户反馈**："这个版本的报告专业度甚至不如V4版本"

## 根因

- 子智能体有 token/context 限制，无法持有一份 2000 行报告的完整上下文
- 子智能体倾向于"总结"而非"展开"，丢失深度分析和数据表格
- 子智能体写作风格不稳定，可能偏离投行专业风格
- 用户对"终极版"期望最高，但子智能体输出质量最差

## 正确做法

| 角色 | 负责 | 禁止 |
|------|------|------|
| delegate_task 子智能体 | Phase 1 数据采集 + Phase 2 Web 研究搜索 | ❌ 写最终版报告 |
| 主智能体（你） | Phase 3 报告定稿 (execute_code + patch) | ❌ 串行采集20+API |

操作模式：
1. `read_file` 读入已有最佳版 MD
2. `re.sub` 清理版本标记
3. `str.replace` 在关键位置插入新章节
4. `write_file` 写回
5. 所有操作在 `execute_code` 中一次完成

## 子智能体正确用法

```python
# ✅ 正确：数据采集
delegate_task(goal='采集所有 22 个 API 数据，返回结构化 JSON', 
              context='调用清单 + 参数', toolsets=['web','terminal','file'])

# ✅ 正确：Web 研究
delegate_task(goal='搜索国际竞争对手和可比交易案例',
              toolsets=['web'])

# ❌ 错误：写最终报告
delegate_task(goal='写终极版投资研究报告...',  # ← 禁止
              context='全部数据 + 37章结构...')
```

## 用户反馈整合

用户期望：
- "不需要每个版本之间的比对" — 报告是独立完整文档
- "不要有任何潜在的或者明显的不同版本的表现" — 零版本痕迹
- "读起来像一次写成的完整报告" — 结构统一、语言一致
