# 首发深度强制令 — 中科江南失败案例

## 背景

2026-06-09，使用 `enterprise-deep-dive-report` v4.0.0 生成北京中科江南（301153.SZ）投资研究报告。

## 失败表现

| 指标 | v4 产出 | 凌度基准 | 差距 |
|------|:--:|:--:|:--:|
| 总行数 | 857 | 2,053 | -58% |
| Web搜索 | **0次** | 5+次 | 全部缺失 |
| Doris DAAS | **0表** | 5+表 | 全部缺失 |
| 可比交易法 | ❌ 无 | ✅ 有 | 缺失 |
| TAM/SAM/SOM | ❌ 无 | ✅ 有 | 缺失 |
| 产业链利润分配 | ❌ 无 | ✅ 有 | 缺失 |
| 管理层能力矩阵 | ❌ 无 | ✅ 有 | 缺失 |
| 专利IPC聚类 | ❌ 无 | ✅ 有 | 缺失 |
| 政府补贴明细 | ❌ 无 | ✅ 有 | 缺失 |

## 根因

技能 v4.0.0 中，Web 搜索和 Doris DAAS 被标记为"V3 增强 / V4 增强"（可选增强章节），描述为"需要投行级深度分析时"才触发。实际执行时，主智能体只跑了 Stage 1A（MCP API 32类），跳过了 Stage 1B（Web搜索）和 Stage 1C（Doris DAAS），导致报告只有基础工商/IP/财务数据，深度严重不足。

## 修复（v5.0.0）

1. Phase 1 拆分为三个强制阶段：1A(MCP) + 1B(Web≥5次) + 1C(Doris≥5表)
2. 新增 Phase 0 深度门禁检查
3. 交付前自动检查：行数<1,800 → 拒绝交付
4. 删除"渐进升级模式"和"先薄后厚"描述
5. "V4增强章节"重命名为"全维度必备章节"

## 教训

> **技能文件中的"可选"和"增强"措辞会被 AI 解释为"可以跳过"。** 如果某个数据维度是报告质量的关键，必须在技能文件中使用"强制""必含""标配"等措辞，并配合交付前门禁检查。
