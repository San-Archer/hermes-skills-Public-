# 报告最终呈现规则 — 版本痕迹清理清单

## 背景

企业深度报告在多次迭代（V1→V2→V3→V4→V5→V6）后，容易携带版本迭代的"伤疤"——"新增章节A"、"补充模块"、"升级内容"等标记。这些标记在最终交付时必须全部清除，让报告读起来像一次性写成的完整投行分析。

## 必须清除的标记

| 模式 | 示例 | 正则 |
|------|------|------|
| 版本号 | V3, V4, V5.0 | `V\d[\.\d]*` |
| 新增/补充章节 | "新增章节A"、"补充章节B" | `新增章节\w` / `补充章节\w` |
| 增强/升级 | "增强模块"、"升级内容" | `增强` / `升级` |
| 字母编号 | A.1, B.2, C.3, D.1 | `[A-Z]\.\d+` |
| 装饰性分隔 | "═══" | 整行删除 |
| 日期重复 | 第二处"报告日期" | 保留第一处 |
| TOC标记 | "-v4新增"、"★新增§" | 从目录中删除 |
| 更新说明 | "## 📑 更新说明" | 整段删除 |
| 元注释 | "本节为V3版本新增内容" | 整句删除 |

## 批量清理脚本

```python
import re

fixes = [
    (r'# 新增章节\w[：:]?\s*', '# '),
    (r'## 补充章节\w[：:]?\s*', '## '),
    (r'【新增】', ''),
    (r'（新增）', ''),
    (r'新增模块', '模块'),
    (r'增强章节', '章节'),
    (r'V\d[\.\d]*\s*', ''),
    (r'\*\*版本新增[：:].*?\*\*', ''),
    (r'\*\*升级内容[：:].*?\*\*', ''),
    (r'（初版[^）]*）', ''),
    (r'^#\s*$', ''),
    (r'\n{4,}', '\n\n\n'),
    (r'-\s*v\d新增', ''),
    (r'★新增.*?\n', ''),
]

for pattern, repl in fixes:
    flags = re.MULTILINE if pattern == r'^#\s*$' else 0
    content = re.sub(pattern, repl, content, flags=flags)
```

## 结构要求

- 章节连续编号：1→N，中间无跳跃
- 无字母编号介入
- 附录为最后一章
- 免责声明在附录最后
- 目录、标题、正文三者编号一致
- 同一个标题模式贯穿全文，不要混用"第X章"和"X."两种格式

## 验证清单

- [ ] 全文搜索"新增"，仅保留商业含义（如"新增装机"、"新增客户"）
- [ ] 全文搜索"V\d"，无匹配
- [ ] 全文搜索"补充/增强/升级"，无结构标记
- [ ] TOC与正文标题一一对应
- [ ] "报告日期"仅出现一次
- [ ] 无装饰性分隔符残留
- [ ] 附录从最后几章开始，ID正确
